     Назва проєкту: «Програмна система для контролю складських приміщень за допомогою RFID міток».  Також проєкт має власну назву: «Anware» що походить від “Anarchy Warehouse”
     
     1.1.  Background
     У сучасному світі логістика та управління запасами є ключовими елементами успіху для підприємств, особливо для тих, що мають великі склади та оперують тисячами товарів одночасно. Зі зростанням обсягів товарообігу та складських площ, ручний контроль товарів стає неефективним і схильним до помилок. Це призводить до втрати часу, неправильного розташування товарів, помилок при інвентаризації та збільшення витрат на управління складом.
     Зростаюча потреба в оптимізації складських процесів породжує необхідність у нових технологічних рішеннях, що дозволяють автоматизувати облік і переміщення товарів. Використання технологій дає можливість значно спростити роботу зі зчитуванням та обліком товарів, дозволяючи уникати людського фактору та підвищити ефективність складу.
     
     1.2.  Business Opportunity
     Основна бізнес-проблема, яку вирішує цей проект, — це необхідність більш ефективного та точного управління запасами на великих складах, де товари переміщуються часто і швидко. Ручний облік призводить до втрат товарів, неправильного розміщення та великих витрат на інвентаризацію. 
     Унікальність цього рішення полягає в тому, що воно забезпечує комплексну автоматизацію інвентаризації, яка буде найдешевша на ринці. Система дозволяє відстежувати рух товарів по секціям і майже миттєво оновлювати інформацію про запаси, що є привабливим для підприємств, які прагнуть підвищити продуктивність, зменшити витрати та уникнути втрат через людський фактор.
     Порівняння існуючих рішень: 
     
     
     Softeon
     Переваги: Досвід роботи понад 20 років, хороша інтеграція з автоматизацією вантажно-розвантажувальних робіт, відносно низька загальна вартість володіння.
     Недоліки: Система менш адаптована до інтеграції з IoT, що обмежує можливості для реального часу та відстеження на складі.

Oracle
     Переваги: Хмарна система з самого початку, SaaS-модель для малого і середнього бізнесу, популярна серед невеликих складів.
     Недоліки: Складно обробляє великі обсяги даних, що створює проблеми для великих складів; хмарний підхід не завжди підходить для локальної інтеграції.

Microsoft Dynamics 365
     Переваги: Гнучка система з можливостями для управління замовленнями, ланцюгами поставок та електронною комерцією.
     Недоліки: Надмірно комплексна і складна для компаній, які потребують лише вузькоспеціалізованих складських операцій; дорогий і складний процес інтеграції.

SAP S/4HANA
     Переваги: Висока стабільність і продуктивність для великих складів з комплексними бізнес-процесами.
     Недоліки: Одна з найдорожчих WMS на ринку, що робить її менш доступною для малого і середнього бізнесу.

Manhattan Associates
     Переваги: Добре підходить для складів дистриб'юторів та роздрібних компаній, сильні позиції в управлінні логістикою.
     Недоліки: Обмежена гнучкість для специфічних потреб великих складів, слабка інтеграція з IoT.

Fishbowl
     Переваги: Легко масштабується для зростання бізнесу, ідеально підходить для базових складських операцій.
     Недоліки: Призначена для малих і середніх компаній; обмежені можливості для великих складів з високим рівнем автоматизації.

Infor SCM/CloudSuite
     Переваги: Широкі можливості для управління складами різного масштабу, популярність серед компаній різного розміру.
     Недоліки: Для великих складів може потребувати додаткових налаштувань і адаптації, не завжди стабільна робота в реальному часі.
     
     Основною нашою перевагою є вартість розробки, впровадження та обслуговування нашого рішення бо вона значно нижча, ніж у глобальних конкурентів. Українським компаніям немає потреби оплачувати дорогі ліцензії або міжнародні послуги підтримки, що значно знижує сукупну вартість володіння системою. Це робить рішення доступнішим навіть для бізнесів з обмеженими бюджетами.

     4.3. Operating Environment
     Web-інтерфейс адміністратора розроблений на VueJS і забезпечує оновлення даних у реальному часі, що дозволяє адміністраторам стежити за поточними змінами та інтегрувати візуалізацію даних. Мобільний додаток для працівників складу, створений на Kotlin з використанням Jetpack Compose для Android. Сервер системи розроблений на Kotlin із використанням фреймворку SpringBoot для побудови REST API та MQTT API. База даних буде базуватися на СУБД MySql. СУБД та серверна частина будуть розгорнуті на Oracle Cloud. На рівні IoT-системи використовуються пристрої зчитування RFID, побудовані на основі ESP8285 NodeMCU, які працюють на спрощеному C++ та передають дані до серверу через протокол MQTT.
