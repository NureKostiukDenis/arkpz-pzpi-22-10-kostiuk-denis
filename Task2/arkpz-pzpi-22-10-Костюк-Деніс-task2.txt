      1. Інженерні рішення
      У проекті для управління користувачами використовується Firebase Authentication та Firebase Realtime Database. Це рішення забезпечує надійний та безпечний механізм аутентифікації та збереження даних користувачів, що дозволяє уникнути розробки власного механізму керування користувачами. 
      Для авторизації в системі використовується UID (унікальний ідентифікатор користувача), який Firebase генерує автоматично під час реєстрації. UID додається до запитів. Це впроваджує безпеку даних, що тільки робочий на складі може використовувати його данні. 
      Також у проєкті використовується дві різні технології для обміну даними, оптимізовані під специфічні задачі. RESTful API використовується для взаємодії між клієнтськими додатками та серверною частиною системи. Для обміну даними між IoT-пристроями (RFID-воротами) та серверною частиною використовується протокол MQTT. Цей протокол був обраний через його оптимізацію для роботи в умовах низької пропускної здатності мережі та мінімального споживання ресурсів.
