Харків 2024
      1. Інженерні рішення
	 Для розгортання API на сервері було використано технологію Docker. Це дозволило створити контейнер, який включав усі необхідні залежності та конфігурації для коректної роботи застосунку. Використання Docker значно спростило процес налаштування середовища, оскільки всі необхідні компоненти були зібрані в одному образі. Завдяки цьому вдалося уникнути проблем, пов’язаних із несумісністю середовищ розробки та сервера. Крім того, Docker забезпечив ізоляцію додатка, що зменшило ризики конфліктів із іншими службами, які працюють на сервері. Це рішення також підвищило гнучкість у масштабуванні та оновленні застосунку, оскільки нові версії можна швидко створювати і запускати у вигляді оновлених контейнерів. 
	Також для розгортання було використано Ngrok для створення тунеля на локальній машині. Це дозволило створити тунель по якому можна звертатися до API
